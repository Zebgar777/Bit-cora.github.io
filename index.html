Unidades Responsivas

Frameworks Unidades Responsivas Mobile First Modulo Grid @Supports

Ideas Clave


1.-Unidades absolutas:
Ejemplos: px (pixeles), cm (centímetros), in (pulgadas). Representan tamaños fijos que no cambian según el contexto. Son útiles para diseños que requieren precisión exacta.
2.-Unidades relativas:
Ejemplos: %, em, rem. Se adaptan según otros elementos o contextos. El % depende del tamaño del contenedor padre. em depende del tamaño de fuente del elemento padre. rem depende del tamaño de fuente del elemento raíz (html).
3.-Unidades de vista de ventana:
Ejemplos: vw, vh, vmin, vmax. Estas unidades son relativas al tamaño de la ventana gráfica del navegador, lo que significa que cambian dinámicamente en función de las dimensiones de la ventana en la que se visualiza el contenido.



Notas


Las unidades absolutas, como px (pixeles), cm (centimetros) e in (pulgadas) representan tamaños fijos y no cambian independientemente del contexto en el que se utilicen, proporcionando una medida exacta que puede ser util para diseños que requieren precision milimetrica.
Las unidades relativas como % (porcentaje) em y rem son flexibles y se adaptan en funcion de otros elementos. Por ejemplo % es relativo al tamaño del contenedor padre, em es relativo al tamaño de fuente de elemento padre y rem es relativo al tamaño de fuente al elemento raiz (html). Las unidades de vista de ventana son herramientas que se ajustan dinamicamente al tamaño de la ventana grafica del navegador. ejmplo: vw, vh, vmin y vmax.



Volver al inicio


Resumen

Las unidades absolutas (como px, cm, in) son fijas y no se ajustan al contexto, lo que las hace útiles para diseños que requieren precisión exacta. Por otro lado, las unidades relativas (% , em, rem) dependen de otros elementos: el % se basa en el tamaño del contenedor padre, em en el tamaño de fuente del elemento padre, y rem en el tamaño de fuente del elemento raíz (html). Las unidades de vista de ventana (vw, vh, vmin, vmax) se ajustan dinámicamente al tamaño de la ventana gráfica del navegador, lo que permite crear diseños responsivos: vw es el 1% del ancho, vh el 1% de la altura, vmin es el valor menor entre ancho y altura, y vmax es el mayor de ambos. Estas últimas son especialmente útiles para diseños fluidos que deben adaptarse a diferentes tamaños de pantalla.

CSS Grid Layout: En el contexto de CSS, Grid Layout es una técnica moderna para crear diseños web complejos.


Notas


El módulo Grid es una de las herramientas fundamentales dentro de los sistemas de diseño web y desarrollo de interfaces. Generalmente, el término "Grid" hace referencia a un sistema de rejilla o cuadrícula, y es utilizado para alinear y distribuir los elementos de una página web de manera ordenada y flexible. En el contexto de CSS (Cascading Style Sheets), el CSS Grid Layout es una técnica moderna y potente para crear diseños complejos en la web.



Volver al inicio


Resumen

El módulo Grid es una herramienta clave en el diseño web moderno, utilizada para estructurar y distribuir los elementos de una página de forma ordenada y flexible. Este sistema de rejilla permite alinear los contenidos de manera eficiente, facilitando la creación de interfaces complejas. En CSS, el CSS Grid Layout es una técnica avanzada que permite crear diseños de múltiples columnas y filas sin depender de soluciones como las flotantes o las tablas.

@Supports

Frameworks Unidades Responsivas Mobile First Modulo Grid @Supports

Ideas Clave


1.-@supports en CSS: Es una regla que permite detectar si un navegador soporta una característica CSS específica, y aplicar estilos condicionales según esa compatibilidad. Esto ayuda a garantizar que los navegadores que no soportan ciertas características aún presenten un diseño funcional.
2.-Mejora progresiva: Es una estrategia de diseño web que consiste en ofrecer una versión básica de la funcionalidad para todos los navegadores y luego agregar características avanzadas para aquellos navegadores que las soporten.
3.-Degradación elegante: Se refiere a la estrategia de crear una funcionalidad avanzada y asegurarse de que se degrade de manera aceptable en navegadores más antiguos o menos capaces, garantizando que la experiencia de usuario sea lo mejor posible en todos los casos.


Notas


La regla @supports en CSS permite detectar si el navegador soporta una caracteristica CSS especifica y aplicar estilos condicionales basados en esta compatibilidad. Es util para implementar funcionalidades modernas mientras se garantiza que los navegadores que no soportan estas caracteristicas aun presenten un diseño funcional.
Mejora progresiva:
La mejora progresiva es una estrategia de diseño web que consiste en crear una version basica de la funcionalidad para todos los navegadores y luego agregar caracteristicas mejoradas para navegadores que la soporten.
Degradacion elegante:
La degradacion elegante por otro lado, se refiere a la estrategia de crear la version mas avanzada de una funcionalidad y asegurarse de que se degrade de manera aceptable en navegadores mas antiguos o menos capases.



Volver al inicio


Resumen

La regla @supports en CSS permite aplicar estilos condicionales dependiendo de si un navegador soporta una característica CSS específica, lo que garantiza que los navegadores incompatibles mantengan un diseño f uncional. En diseño web, la mejora progresiva busca ofrecer una funcionalidad básica para todos los navegadores y luego agregar características avanzadas para los que las soporten. Por otro lado, la degradación elegante se enfoca en crear una versión avanzada de una funcionalidad que se degrade de forma aceptable en navegadores más antiguos o limitados.
